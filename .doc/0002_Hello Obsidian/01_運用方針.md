# Obsidian 運用方針（vault ディレクトリ）

- 管理対象: `vault/` ディレクトリを Obsidian の Vault として管理します。
- 保存形式: 拡張子は `.md`（UTF-8）、標準 Markdown を基本とします。
- メタデータ: Obsidian の `.obsidian/`（ワークスペース設定や一時ファイル）は Git では無視します（リポジトリ直下の `.gitignore` で除外済み）。
- 添付ファイル: `vault/_assets/` 配下に配置推奨（画像・PDF 等）。
- リンク方針: 相対リンク（例: `![[./_assets/image.png]]` または `![](./_assets/image.png)`）を推奨。
- 命名規則: 先頭に通番 + スラッグ、または日本語タイトルで OK（プロジェクトで統一）。
- バックアップ: 重要ノートは通常の Git コミットに含まれます（Obsidian 設定は除外）。

## 使い方

1. Obsidian を起動 → 既存のフォルダを開く → リポジトリの `vault/` を選択。
2. 必要に応じてプラグインを有効化（コミット対象外）。
3. ノートは `vault/` に作成。添付は `vault/_assets/` に保存。

## 既存ドキュメントとの関係

- 現在の開発系ドキュメントは `.doc/` 配下にあります。
- `.doc/` を Obsidian からも参照したい場合は、`vault/` 直下に `doc/` という名前でミラー（コピー）を作る、または運用で `vault/` へ順次移行してください。
- 自動同期（スクリプトやシンボリックリンク）は要望があれば提案します。

## 今後の拡張（任意）

- ブログ連携: `vault/` から Next.js にエクスポートしてブログ記事化（MD/MDX 取込）のパイプラインを後で追加可能。
- 共有設定: `.obsidian/` 内の一部設定のみ共有したい場合は、対象ファイルを限定して `.gitignore` を調整します。
